Github is a cloud like dropbox, but for code. Integrating the use of github in your quant projects can save you a lot of time and space on all your other clouds while you change between your workplaces and computers.

This article is for my personal memory only. There are some good git(hub) tutorials on youtube if you want a more detailed guide on github.

Setting up Git for the first time on a new project is fairly simple, you just need to enter these five commands into your terminal. Be sure to be sitting in the directory where your project is located!

You only need to do this once, right when you start your new project. If you create a completely new project, you’ll have to go through these commands again. But only once per project…


$ git config --global user.name "Your Name"
$ git config --global user.email "you@youraddress.com"
$ git config --global push.default matching
$ git config --global alias.co checkout
$ git init

After this setup in the Git Bash terminal go to github.com and login.  From create a 'New Repository':

This will bring you to the following page which shows how to finally upload your files to the github cloud.

You don't need to run every command like 'git add README.md'. It's just best practice to do so.


Running these commands should be sufficient:
git add .
git commit -m 'initial commit'
git remote add origin *your link to the repository'
git push -u origin master

Youcan run these commands every time before and after you are doing major changes to yourcode and push all files to github. Your code is now accessible on github from everywhere and your dropbox space will thank you.
