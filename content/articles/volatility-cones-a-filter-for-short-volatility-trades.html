When you sell options (i.e., go short volatility), you’re implicitly making a forecast that the option implied volatility is overstated and realized volatility will be lower. But “hope” isn’t a strategy—historical context is. This is where the volatility cone comes in. It shows you how low or high realized volatility has run in the past for various time horizons (e.g. 20 days, 40 days, 120 days).

If implied volatility is currently at the higher end of that historical range (for a given horizon), it might be a prime spot to sell options. If the implied is stuck in the middle (or near historical lows) while there’s a big Fed event looming, you might reconsider. The volatility cone is basically a quick filter—it doesn’t guarantee profit, but it tells you whether the current implied volatility environment is historically “cheap” or “expensive.”

Let’s jump into the code. We start by importing the necessary libraries and downloading historical data for the Lulu Lemon stock:


import yfinance as yf
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

ticker = "LULU"
end_date = "2025-01-02"
start_date = "2021-01-02"

df = yf.download(ticker, start=start_date, end=end_date)
df.dropna(subset=["Close"], inplace=True)

Next, we calculate log returns from the closing prices:


df["LogRet"] = np.log(df["Close"] / df["Close"].shift(1))
df.dropna(subset=["LogRet"], inplace=True)

Our goal is to look at rolling annualized volatility over various windows, such as 20, 40, 60 days, and so on. An advantage of rolling windows is that we get a lot of volatility values from our dataset, but as these values differ only slighty due to the nature of rolling windows (or moving averages) this can really underestimate the volatility. To account for this Hodges & Tompkins (2002) propose a correction factor m_{\mathrm{var}} for the variance. For volatility (std. dev.), multiply by \sqrt{m_{\mathrm{var}}}. See their paper for more details.


def hodges_tompkins_factor(h, total_obs):
    n = total_obs - h + 1
    if n <= 0:
        return 1.0
    denom = 1.0 - (h / n) + (h**2 - 1.0) / (3.0 * n**2)
    if denom <= 0:
        return 1.0
    return 1.0 / denom

The function above calculates the adjustment factor for a variance proposed by Hodges and Tompkins for a given window length and the number of total observations. As we are concerned about volatility we will take the square root of this factor.


windows = [20, 40, 60, 120, 240]
total_obs = len(df["LogRet"])
vol_data = {}

for w in windows:
    ht_var = hodges_tompkins_factor(w, total_obs)
    ht_vol = np.sqrt(ht_var)
    
    rolling_vol = (
        df["LogRet"]
        .rolling(window=w)
        .std()
        .dropna()
        * np.sqrt(252)
        * ht_vol
    )
    vol_data[w] = rolling_vol.values

The code above loops over all window length of interest, calculates the adjustment factor and calculates annualized rolling volatiliy values for LULU from 2021 to 2025. These values are stored as numpy arrays for each window in the dictionary vol_data.

To form the "cone," we typically gather five summary statistics: minimum, 25th percentile, median, 75th percentile, and maximum volatility, then plot them for each window side by side.

The stats_df DataFrame now contains one row per rolling window (20, 40, 60, etc.) and columns for "Min", "Q1", "Median", "Q3", and "Max". Let's create a plot out of that!


plt.figure(figsize=(8,4))

plt.plot(stats_df.index, stats_df["Max"],    label="Max", color="darkgray")

plt.title("Volatility Cone for " + ticker)

plt.ylim(0, 1)  # adjust the y-limit if needed
plt.legend()
plt.grid(True)
plt.show()

 

And that’s it! We’ve created a “volatility cone” that shows how realized annualized volatility for different rolling windows has varied over the past few years. Use these ranges to gauge if today’s implied volatility is historically “expensive” or “cheap” for each horizon, and weigh that against upcoming events or changes in market conditions. For example, if you’re considering selling a 1‐month option, you look at the 1‐month segment of the cone (the 20‐day stats). If the market’s implied volatility for the next 20 days is at the 80th percentile relative to historical realized distribution, that’s potentially favorable for a short vol position—assuming you believe realized vol will revert.
